# FINNTRAIL homepage markup

Статическая верстка главной страницы FINNTRAIL для последующей интеграции в 1С-Битрикс.

## Стек

- HTML
- SCSS/Sass
- JavaScript ES6+
- Vite только для локальной разработки и сборки статических ассетов

CSS-фреймворки и SPA-фреймворки не используются.

## Команды

```bash
npm install
npm run dev
npm run build
```

Сборка складывается в `dist/`.

## Структура

- `index.html` - разметка главной страницы с комментариями для Bitrix.
- `scss/main.scss` - точка входа стилей.
- `scss/components/` - SCSS по компонентам.
- `js/main.js` - интерактивность главной страницы.
- `images/icons/sprite.svg` - SVG-спрайт иконок.
- `images/content/` - временные контентные изображения для статической верстки.
- `fonts/` - локальные шрифты.

## Интеграция в Bitrix

Разметка разделена на зоны:

- `header.php`: верхняя навигация до `<main>`.
- шаблон главной страницы: секции внутри `<main class="page-content">`.
- `footer.php`: футеры и мобильный таббар после `</main>`.

Комментарии `BITRIX:` в `index.html` указывают предполагаемые компоненты и динамические места. Для товарных блоков используется структура, совместимая с `bitrix:catalog.section`; для меню и футера - `bitrix:menu`; для корзины в шапке - `bitrix:sale.basket.basket`.

## Изображения

В статической верстке используются исходные JPG/PNG/SVG из `images/content/`. По ТЗ конвертацию в WebP/AVIF и ресайзы выполняет серверная часть CMS.

При интеграции нужно сохранить атрибуты:

- `src`
- `srcset`
- `sizes`
- `loading="lazy"` для изображений ниже первого экрана

Текущие `srcset` в статике являются точками подстановки. На стороне Bitrix их нужно заменить на реальные наборы размеров, например:

```html
srcset="/upload/.../image-480.webp 480w, /upload/.../image-960.webp 960w, /upload/.../image-1440.webp 1440w"
sizes="(max-width: 767px) 100vw, (max-width: 1279px) 50vw, 33vw"
```

Hero-видео должно оставаться с `preload="metadata"` и `poster`.

## Интерактивные состояния

В компонентах заложены классы состояний:

- `is-active`
- `is-inactive`
- `is-loading`
- `is-visible`
- `is-kit-open`

При интеграции форм и AJAX-сценариев нужно дополнить серверными/JS-состояниями:

- `disabled`
- `loading`
- `error`
- `success`
- validation error
- empty result
- retry

## Проверка перед передачей

1. `npm run build`
2. Проверить консоль браузера на отсутствие критических ошибок.
3. Проверить адаптивы: 320-767, 768-1279, 1280+.
4. Проверить, что динамические изображения ниже первого экрана имеют lazy loading.
5. Проверить, что Bitrix-интеграция заменяет статические пути и `srcset` на CMS-генерируемые.
